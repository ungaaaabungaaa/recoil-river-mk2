# UI direction

## Visual language

Use black, white, and grey. Build dense monochrome knowledge spheres from original technical line art. Pair bold typography with short labels and oversized rounded surfaces. Keep copy sparse and let saved-page imagery, node clusters, and connection paths carry the interface.

Avoid color-coded taxonomies, metric dashboards, decorative gradients, and long explanatory panels. Use contrast, scale, proximity, and line weight to establish hierarchy.

## Surface direction

### Login

The login view places a graph sphere on the left and a rounded light sign-in panel on the right. The graph shows the product’s connected-material idea without claiming that an account has content before sign-in.

### Landing page

The landing page opens with a dark hero, original technical line art, and a light rounded call-to-action block. It explains deliberate capture and private organization in short copy.

### Extension popup

The popup stays compact and graph-first. It shows current-page context, the authenticated account’s local graph preview, and one primary `Add this page` action. The action remains manual.

### Website library and mind map

The library leads with image-backed saved-page cards. The mind map fills the workspace with an Obsidian-style node field and visible relationships. These are two views of one private account library.
